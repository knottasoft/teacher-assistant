# Демо-данные оценок для мастер-класса педсовета «Машук-2026»

**Назначение:** материалы для Акта 5 («ИИ работает фоном») мастер-класса Артёма Маршалкина — демонстрация навыка `/grade-report` и MCP-инструмента `grade_analytics` плагина `teacher-assistant`.

**Происхождение данных:** все ФИО, оценки и аномалии **синтетические** — сгенерированы python-скриптом `gen_grades.py` (seed = 42, воспроизводимо). К реальным ученикам какой-либо школы данные отношения не имеют.

## Контекст

- Учебный год: 2025–2026
- Период: II четверть (07.11.2025 – 27.12.2025)
- Класс: 8А / 8Б / 8В (синтетика)
- Предмет: Литература
- Программа: на основе реальной рабочей программы 8 кл., автор Перекрестова Т. Н., МКОУ «Малодербетовская СОШ им. К. Д. Убушиевой» (программа лежит рядом — `артефакты/рабочая_программа_8_кл_литература_2023-2024.pdf`)
- Темы II четверти (по КТП): А.С. Пушкин «История пугачёвского бунта», «Капитанская дочка» (8 уроков + 2 кл. сочинения), стихотворения; М.Ю. Лермонтов «Мцыри» (4 урока)
- Учитель: Перекрестова Татьяна Николаевна (как в исходной программе)

## Структура файлов

В папке два формата на каждый класс — итого **6 CSV-файлов** + этот README.

### `grades_8X_literature_q2_full.csv` — реалистичный экспорт журнала

Кодировка UTF-8-BOM, разделитель — точка с запятой (как в большинстве экспортов МЭШ / Дневник.ру / Сетевой Город / ЭлЖур для совместимости с русским Excel).

Структура соответствует типовому формату экспорта российских электронных журналов:

```
[метаинформация: ОУ, класс, предмет, учитель, период]

[пустая строка]

№ | ФИО | дата1 (тип работы) | дата2 (тип работы) | ... | За II четверть
   |     | тема урока 1       | тема урока 2       | ... |
1 | Ученик 8А-01 | 5 | 4 | ... | 5
2 | Ученик 8А-02 | ...

[легенда типов работ + условные обозначения]
```

Типы работ: `Т` — текущий ответ, `ДЗ` — домашнее задание, `СР` — самостоятельная, `ПР` — проверочная, `КС` — классное сочинение, `ЧН` — чтение наизусть.

Условные обозначения: `Н` — отсутствие, `Б` — болезнь, `н/а` — не аттестован.

**Этот файл показываем зрителю** как «вот что учитель выгружает из журнала одной кнопкой».

### `grades_8X_literature_q2_simple.csv` — упрощённый формат для `grade_analytics`

Кодировка UTF-8, разделитель — запятая. Без метаинформации, без дат, без типов работ. Только:

```
ФИО,оценка_1,оценка_2,...,оценка_N
Ученик 8А-01,5,4,5,4,5,...
```

Этот формат соответствует тому, что MCP-инструмент `grade_analytics` плагина `teacher-assistant` фактически парсит (см. `mcp-server/src/tools/grade-analytics.ts`). Все нечисловые значения (`Н`, `Б`, пусто) уже отфильтрованы — в файл попали только баллы 2–5.

**Этот файл подаём на вход** `grade_analytics`.

## Заложенные в данные сюжеты для разбора

Чтобы плагин/ИИ имели что показать в аналитике, в данные намеренно встроены три сюжета — то, что должно «вылавливаться» в `students_at_risk`, `top_students`, `distribution`:

### 8А — благополучный класс
- Средний балл ~ 4.1
- 5–6 отличников, 1–2 ученика в группе риска
- Один отличник (Ученик 8А-05) пропустил оба классных сочинения по болезни — отдельный кейс «надо догнать»
- Один слабый ученик (Ученик 8А-24) — высокая частота пропусков

### 8Б — проблемный класс
- Средний балл ~ 3.5
- 5–6 в группе риска, всего 1–2 отличника
- **Главный сюжет**: четыре ученика (Ученик 8Б-13, 8Б-14, 8Б-15, 8Б-16) показывают одинаковый паттерн — после 7-го урока (рубеж первого классного сочинения) резкий провал оценок с 4–5 до 2–3. На этой аномалии можно строить педагогическую гипотезу: «общая причина — изменение условий обучения, например, переход к одному репетитору, переезд на новое расписание, конфликт с одноклассниками после контрольной» (это и есть «то, что говорят данные» — кейс из конспекта обсуждения 21.04.2026).

### 8В — смешанный класс
- Средний балл ~ 3.8
- 3 отличника, 3 в группе риска
- **Главный сюжет**: ученица 8В-13 в начале четверти троечница (3-3-4-3-3), к середине подтянулась (4-4-4-3-4), к концу — 5-5-5-5. Это классический пример «прогресса» — должен попасть в `top_students` или быть подсвечен ИИ как «положительная динамика».

## Как использовать в Акте 5 мастер-класса

**Эпизод 5.1 (фоновая задача по расписанию):** показываем `grades_8a_literature_q2_simple.csv` и говорим: «допустим, каждое воскресенье в 21:00 ИИ забирает свежий экспорт из электронного журнала, прогоняет `/grade-report`, кладёт PDF/MD-отчёт в папку класса». Запускаем сами скилл прямо в Cowork.

**Эпизод 5.2 (аналитика без формул):** на сцене подаём `grades_8b_literature_q2_simple.csv` (проблемный класс) → MCP `grade_analytics` выдаёт средний балл / качество / группу риска / распределение / top-учеников. Реплика: «обратите внимание на этих четырёх. Они идут вместе — общая причина? Это вопрос для классного руководителя».

## Воспроизведение

```
python3 gen_grades.py
```

Скрипт детерминирован (`random.seed(42)`), переименование классов / правки распределения уровней — внутри.
