# Генерация задач по физике

- **Класс:** $ARGUMENTS[0]
- **Тема:** $ARGUMENTS[1]
- **Количество:** $ARGUMENTS[2] (по умолчанию: 8)

## Требования

1. Задачи трёх уровней:
   - **Качественные** (30%) — объяснение явления, без вычислений
   - **Расчётные простые** (40%) — 1-2 действия, прямое применение формулы
   - **Расчётные сложные** (30%) — 3+ действий, комбинация формул

2. Для каждой задачи:
   - Условие с корректными единицами СИ
   - Решение по схеме: Дано → Найти → Решение → Ответ
   - Проверка размерности
   - Указание используемых формул и законов

3. Правила:
   - Реалистичные числовые значения
   - g = 9,8 м/с² (допустимо 10 м/с² для упрощения)
   - Для графических задач — описание графика/схемы
   - Единицы СИ (при необходимости — перевод в условии)

4. Для 9 класса — задачи в формате ОГЭ
5. Для 11 класса — задачи в формате ЕГЭ

6. Сохрани:
   - `задачи_физика_[класс]_[тема].md` — условия
   - `задачи_физика_[класс]_[тема]_решения.md` — решения

Справочник формул: [formulas-reference.md](formulas-reference.md)

## Self-validation перед завершением

Перед тем как сообщить пользователю «готово», проверь, что выходной файл содержит:

- [x] **Класс** (например, «5 класс», «8 класс»)
- [x] **Предмет** (русский язык, математика, физика, литература)
- [x] **Привязка к нормативной базе:** ФГОС (приказ Минпросвещения №287/2021 для ООО, приказ №413/2012 в действующей редакции для СОО), ФОП (№370 или №371 от 2023), либо ФРП от 2024
- [x] **Планируемые результаты** или **УУД** (для планов уроков, заданий, КТП)
- [x] **Тип документа** в заголовке (план урока / задание / тест / отчёт / КТП)

Если чего-то не хватает — допиши до возврата управления, не возвращай учителю недооформленный документ. Это правило заменяет общий PostToolUse-hook в плагине и относится только к выходу этого skill'а.
