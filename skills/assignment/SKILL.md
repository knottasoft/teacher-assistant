# Создание задания

Создай задание:
- **Предмет:** $ARGUMENTS[0]
- **Класс:** $ARGUMENTS[1]
- **Тема:** $ARGUMENTS[2]
- **Тип:** $ARGUMENTS[3] (если не указан — спроси)

## Порядок действий

1. Определи уровень сложности по классу и теме
2. Используй `fgos_lookup` для проверки содержания по стандарту
3. Создай задания трёх уровней сложности:
   - **Базовый** (60% заданий) — знание и понимание
   - **Повышенный** (30% заданий) — применение и анализ
   - **Высокий** (10% заданий) — синтез и оценка
4. Для тестов — создай 2 варианта с 4 вариантами ответов
5. Создай ключ ответов с баллами за каждое задание
6. Рассчитай шкалу оценивания (баллы → оценка по 5-балльной системе):
   - «5» — 90-100% баллов
   - «4» — 70-89% баллов
   - «3» — 50-69% баллов
   - «2» — менее 50% баллов
7. Сохрани файлы:
   - `задание_[тема]_вариант1.md`
   - `задание_[тема]_вариант2.md` (для тестов/контрольных)
   - `ключ_[тема].md`

## Self-validation перед завершением

Перед тем как сообщить пользователю «готово», проверь, что выходной файл содержит:

- [x] **Класс** (например, «5 класс», «8 класс»)
- [x] **Предмет** (русский язык, математика, физика, литература)
- [x] **Привязка к нормативной базе:** ФГОС (приказ Минпросвещения №287/2021 для ООО, приказ №413/2012 в действующей редакции для СОО), ФОП (№370 или №371 от 2023), либо ФРП от 2024
- [x] **Планируемые результаты** или **УУД** (для планов уроков, заданий, КТП)
- [x] **Тип документа** в заголовке (план урока / задание / тест / отчёт / КТП)

Если чего-то не хватает — допиши до возврата управления, не возвращай учителю недооформленный документ. Это правило заменяет общий PostToolUse-hook в плагине и относится только к выходу этого skill'а.
