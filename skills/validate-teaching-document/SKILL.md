# Self-validation перед завершением учебного документа

## Overview

Сдача учителю учебного документа без проверки атрибутов — это халтура, не эффективность.

**Core principle:** Класс, предмет, ФГОС-привязка должны быть в файле, а не в чьей-то голове.

## The Iron Law

```
NO TEACHING-DOC COMPLETION CLAIMS WITHOUT FRESH ATTRIBUTE VERIFICATION
```

Если ты не проверил эти 5 пунктов в этом ответе — ты не можешь сказать «готово».

## The Gate Function

Перед тем как сообщить пользователю «готово», скажешь ли ты «вот план урока», «отчёт сохранён», «материал подготовлен» — обязательно:

```
1. ОТКРЫТЬ выходной файл (даже если ты только что его написал)
2. ПРОВЕРИТЬ 5 пунктов:
   ✓ Класс (например, «5 класс», «8 класс», «10-11 класс»)
   ✓ Предмет (русский язык / математика / физика / литература)
   ✓ Привязка к нормативной базе (ФГОС / ФОП / ФРП и конкретный приказ)
   ✓ Планируемые результаты или УУД (для планов уроков, заданий, КТП, отчётов)
   ✓ Тип документа в заголовке (план урока / задание / тест / отчёт / КТП / квиз / раздаточный материал / диктант)
3. ЕСЛИ чего-то не хватает — ДОПИСАТЬ до возврата управления
4. ТОЛЬКО ТОГДА сказать «готово»

Пропустишь шаг = врёшь, не проверяешь.
```

## Common Failures

| Что есть | Что не хватает | Действие |
|---|---|---|
| Полный план урока | Не указан класс | Добавить «**Класс:** 8» в шапку |
| Тест с ключами | Нет привязки к ФГОС | Добавить «**Соответствие ФГОС ООО (приказ Минпросвещения №287/2021)**» |
| Отчёт об успеваемости | Нет указания периода | Добавить «**Период:** II четверть 2025-2026 уч. года» |
| КТП на год | Нет планируемых результатов | Добавить колонку «Планируемые результаты» с УУД |
| Диктант | Нет норм оценивания | Добавить раздел «Нормы оценивания диктанта» |

## Red Flags — STOP

- Используешь «вот», «готово», «как и просили» **без проверки файла**
- Собираешься объявить готовность **по памяти**, не открыв output
- Думаешь «учитель сам допишет класс»
- «План короткий, и так понятно»
- «Ну предмет же из контекста ясен»

Любой из этих сигналов = вернись на шаг 2 в Gate Function.

## Нормативная рамка для подсказок

Когда нужно указать привязку к ФГОС/ФОП — используй актуальные документы (на 2026 г.):

- **ФГОС ООО (5-9 класс):** приказ Минпросвещения России №287 от 31.05.2021
- **ФГОС СОО (10-11 класс):** приказ Минобрнауки России №413 от 17.05.2012 в действующей редакции (последняя — приказ №93 от 12.02.2025)
- **ФОП ООО:** приказ Минпросвещения №370 от 18.05.2023 (в действующей редакции от 08.10.2025)
- **ФОП СОО:** приказ Минпросвещения №371 от 18.05.2023 (в действующей редакции от 10.11.2025)
- **ФРП по литературе:** приказ Минпросвещения №171 от 19.03.2024
- **Лимит контрольных:** приказ Минпросвещения №704 от 09.10.2024 (не более 10% учебного времени)

## When to Apply

**Всегда перед:**
- Сообщением «готово» / «план готов» / «отчёт сохранён»
- Завершением работы skill'а из плагина teacher-assistant
- Сохранением файла через `Write` для учителя
- Передачей результата на DOCX-экспорт

**Не применяется к:**
- Промежуточным черновикам, которые учитель ещё будет править вместе с тобой
- Технической переписке без файла на выходе
- Фрагментам, которые продолжают существовать в чате как обсуждение

## The Bottom Line

**Никаких сокращений.**

Открой файл. Проверь 5 пунктов. ТОЛЬКО ТОГДА — «готово».

Это правило заменяет PostToolUse-hook плагина и применяется ко всем skill'ам teacher-assistant, генерирующим учебный документ.
